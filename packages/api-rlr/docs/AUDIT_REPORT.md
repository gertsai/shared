# Отчет аудита пакета @orchdev/api-rlr

## Дата: 2024

## 1. Обнаруженные проблемы

### 1.1 Ошибки линтинга

#### ✅ Исправлено:

- **src/adapters/ResilientRedisAdapter.ts**: Удален неиспользуемый импорт `Redis`

#### ⚠️ Требует внимания (в тестовых файлах):

- **src/services/RouteResolver.test.ts**: 4 предупреждения о non-null assertion (`!`)
  - Рекомендация: Использовать optional chaining (`?.`) вместо `!`

### 1.2 Неиспользуемые файлы

#### Потенциально устаревшие файлы:

1. **src/scripts/scripts.ts** - Содержит 17 версий incrementSw скриптов (incrementSw1-incrementSw17)
   - Используется только в `client/rlr.ts` для импорта `increment` скрипта
   - Рекомендация: Удалить неиспользуемые версии, оставить только актуальные

2. **src/client/index.ts** - Простой реэкспорт
   - Используется только в `test-utils/RateLimitTestUtils.ts`
   - Можно упростить импорты

### 1.3 Архитектурные проблемы

#### Дублирование функционала:

1. **Два параллельных подхода к Lua скриптам:**
   - Старый: `LuaScriptManager` + прямое использование в `client/rlr.ts`
   - Новый: `TypedLuaScriptManager` + `TypedLuaScript`
   - **Проблема**: Новый подход не интегрирован полностью

2. **Три способа создания адаптеров:**
   - Прямое создание `RedisAdapter`
   - Через `AdapterFactory`
   - `TypedRedisAdapter` (не используется в продакшене)

### 1.4 Логические несоответствия

1. **Несогласованность в стратегиях:**
   - В `RoutePresets` используется `'gcra' as const` вместо `LimiterStrategy.GCRA`
   - Смешивание строковых литералов и enum значений

2. **Неполная интеграция новой архитектуры:**
   - `useModularArchitecture` флаг по умолчанию `false`
   - Новые компоненты (`TypedRedisAdapter`, `TypedLuaScriptManager`) не используются без явной активации
