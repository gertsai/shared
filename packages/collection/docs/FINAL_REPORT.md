## 🔧 Исправленные проблемы

### Критический баг в OrderedMap

- **Проблема**: Класс OrderedMap должен поддерживать порядок вставки элементов, но методы `keys()` и `values()` использовали базовую реализацию из Map, игнорируя связный список
- **Симптомы**:
  - `moveToFront()`, `moveToBack()`, `moveBefore()`, `moveAfter()` не работали корректно
  - Порядок элементов при итерации не соответствовал ожидаемому
- **Решение**: Переопределены методы `keys()` и `values()` для итерации по связному списку
